# Release v0.2 — Testing & Performance

**Status:** Released  
**Scope:** Test suites, benchmarks, first measured optimization

## Features

- [x] Integration test suite (`tests/integration/`)
  - End-to-end sessions: book building, multi-level sweeps, market sells
  - Cancel-after-partial-fill, interleaved add/cancel with FIFO verification
  - Trade quantity conservation across a session
- [x] Randomized stress suite (`tests/stress/`)
  - 10,000 random operations with invariants asserted after every step:
    no crossed book, no empty levels, cached level totals in sync
  - Fixed RNG seed — failures reproduce identically
- [x] Google Benchmark suite (`benchmarks/`)
  - Throughput: add / cancel / best_bid / submit / market sweep
  - Latency percentile harness: p50/p90/p99/p99.9/max per call
  - Allocation counter (operator new/delete override) for hot-path audits
- [x] Optimization: caller-provided trade buffer
  - `submit_order(Order, std::vector<Trade>&)` overload for hot paths
  - Avg allocations per submit: 2.46 → 1.52; 47% of submits allocation-free
  - p50/p90/p99 all improved; documented in benchmark report

## Measured performance (MacBook Air, macOS 12, Release build)

| Operation | Result |
|-----------|--------|
| `best_bid` | 2.6 ns |
| `submit_order` p50 | ~440 ns |
| `submit_order` p99 | ~2.3 us |
| `cancel_order` p50 | ~100 ns (zero heap allocations) |

Full report with methodology and caveats: [docs/benchmarks/report-2026-07-13.md](../benchmarks/report-2026-07-13.md)
