# Release v0.1 — Core Engine

**Status:** Released  
**Scope:** Order Book + Matching Engine + Interactive CLI

## Features

- [x] Domain types (`Order`, `Trade`, `Side`, `Price`, `Quantity`)
  - Strong typedefs prevent type-confusion bugs at compile time
  - Fixed-point prices (integer ticks) — no floating point on the matching path
- [x] Order Book with price levels and FIFO queues
  - `std::map` per side (bids sorted high-to-low, asks low-to-high)
  - O(1) cancel via `std::unordered_map` order index holding list iterators
  - Empty price levels erased eagerly — `best_bid`/`best_ask` always valid
- [x] Matching Engine with price-time priority
  - Limit and market orders, partial fills, multi-level book sweeps
  - Trades execute at the resting order's price (price improvement for the aggressor)
  - Market order remainders discarded — never rest
- [x] Interactive CLI (`mercury_cli`)
  - `buy`/`sell` limit and market orders, `cancel`, live `book` view with spread
  - Decimal-to-tick conversion at the boundary; core never sees floats
- [x] Test suite: 46 tests (unit + integration + randomized stress)
  - 10,000-operation stress run asserting book invariants after every step

## Complexity

| Operation | Cost |
|-----------|------|
| `add_order` | O(log n) levels |
| `cancel_order` | O(1) |
| `best_bid` / `best_ask` / `spread` | O(1) |
